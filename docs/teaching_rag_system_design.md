# 教学知识图谱系统设计方案

## 概述

本方案针对教学场景，在ASTRA框架基础上进行三项核心创新，目标是构建一个**智能化、个性化、自适应**的知识图谱系统。

---

## 第一部分：结构感知的分层节点建立

### 1.1 问题分析

**现状**：ASTRA通过内容提取→实体识别→图构建的**底向上**（bottom-up）流程创建节点。
- 优势：数据驱动，无需外部标注
- 劣势：无法直接利用教材已有的层级结构（章节→小节→知识点）

**教学场景需求**：
- 教材天然具有清晰的结构化组织（目录、章节、小节）
- 学习过程中，学生需要从**多个粒度**理解知识（宏观→微观）
- 前置知识依赖关系通常由教材本身定义

### 1.2 设计方案

#### 1.2.1 元数据驱动的初始化

**输入格式**：教材的结构化元数据（JSON/YAML格式）

```
{
  "title": "机器学习基础",
  "parts": [
    {
      "id": "part_1",
      "title": "第一部分：基础概念",
      "chapters": [
        {
          "id": "ch_1",
          "title": "第1章：监督学习",
          "sections": [
            {
              "id": "sec_1_1",
              "title": "1.1 线性回归",
              "learning_objectives": ["理解线性模型", "掌握最小二乘法"],
              "key_concepts": ["斜率", "截距", "残差"],
              "content_range": [100, 350],
              "estimated_importance": 0.9
            }
          ]
        }
      ]
    }
  ],
  "relationships": [
    {
      "from_id": "sec_1_1",
      "to_id": "sec_1_2",
      "relation_type": "prerequisite",
      "relation_strength": 0.9
    }
  ]
}
```

**处理流程**：

1. **递归层级处理**：自上而下遍历元数据树
   - Level 0：Part（部分）
   - Level 1：Chapter（章）
   - Level 2：Section（节）
   - Level 3：Subsection（小节）
   - Level 4+：细粒度概念

2. **结构节点创建**：为每个元数据节点创建对应的图节点
   - 节点类型：`entity_type = "structural"`
   - 节点字段：`structural_level`、`parent_structural_id`、`estimated_importance`

3. **层级边构建**：相邻层级节点间建立`hierarchical_contains`关系
   - 权重设为1.0（强关联）
   - 标记为`is_structural = True`

#### 1.2.2 前置知识映射

**处理元数据中的显式关系**：
- `prerequisite`：前置知识，必须先学
- `supports`：支撑知识，可选性增强
- `conflicts`：矛盾知识，需要澄清
- `extends`：扩展知识，深度延伸

**边的处理**：
- 直接创建对应的图边
- `relation_strength`字段从元数据继承（或默认0.8）
- 标记为结构性边，用于后续查询时的优先级调整

### 1.3 工作流

```
教材元数据 
  ↓ 验证格式
元数据结构化表示
  ↓ 递归遍历
创建结构节点（Chapter、Section等）
  ↓ 建立层级边
结构骨架完成
  ↓ 内容提取（ASTRA标准流程）
创建概念/细节节点 + 内容边
  ↓ 权重计算
完成加权图
```

## 第二部分：关系权重量化体系

### 2.1 问题分析

**现状**：ASTRA中所有边的`relation_strength`都是0.5（通用值），无法区分强弱关联。

**教学需求**：
- 不同知识点间的关联强度差异大
  - "导数"与"微积分"：强关联（概念包含）
  - "导数"与"泰勒级数"：中等关联（应用拓展）
  - "导数"与"统计学"：弱关联（领域相关）
- 查询时应优先返回**强关联**的知识点
- 学生复习时应有**渐进式深化**的路径

### 2.2 权重量化方法

#### 2.2.1 四分量权重公式

$$W = w_1 \cdot S_{sem} + w_2 \cdot S_{struct} + w_3 \cdot S_{co} + w_4 \cdot S_{exp}$$

其中：

| 分量 | 含义 | 计算方式 | 权重占比 |
|------|------|---------|---------|
| $S_{sem}$ | 语义相似度 | 概念名称+描述的embedding余弦相似度 | 30% |
| $S_{struct}$ | 结构接近度 | 层级树中的距离+同父级关系 | 25% |
| $S_{co}$ | 共现频率 | 同一段落内出现的比例 | 25% |
| $S_{exp}$ | 显式关系 | 元数据中明确指定的关系强度 | 20% |

#### 2.2.2 各分量详解

**语义相似度** $S_{sem}$：
- 计算方法：
  1. 对两个概念的名称分别embedding
  2. 对两个概念的描述分别embedding
  3. $S_{sem} = 0.4 \cdot \cos(e_{name1}, e_{name2}) + 0.6 \cdot \cos(e_{desc1}, e_{desc2})$
- 取值范围：[0, 1]
- 适用场景：跨层级概念的关联

**结构接近度** $S_{struct}$：
- 计算方法：
  1. 同一父节点的两个节点：$S = 1.0$
  2. 父子关系（直接包含）：$S = 0.95$
  3. 距离为$d$的节点：$S = 1.0 / (1 + 0.2d)$（衰减函数）
  4. 距离超过5层：$S = 0.1$
- 取值范围：[0, 1]
- 适用场景：同一教材章节内的概念

**共现频率** $S_{co}$：
- 计算方法：
  1. 在文本中查找两个概念的所有出现位置
  2. 统计在一定窗口内（如200字）同时出现的比例
  3. $S_{co} = \frac{\text{共现次数}}{\min(\text{概念1出现次数}, \text{概念2出现次数})}$
- 取值范围：[0, 1]
- 适用场景：同一教学单元内的概念

**显式关系强度** $S_{exp}$：
- 计算方法：
  1. 优先使用元数据中的`relation_strength`值
  2. 若无显式值，根据`relation_type`使用默认值：
     - `prerequisite`：0.9
     - `supports`：0.7
     - `extends`：0.6
     - `conflicts`：0.4
- 取值范围：[0, 1]
- 适用场景：有明确关系定义的知识点

#### 2.2.3 权重阈值与过滤

**权重阈值** $\theta$：
- 默认值：0.3
- 低于阈值的边不纳入图中（减少噪音）
- 可调参数，根据不同教材调整

**权重分层**：
- $W > 0.8$：强关联（红色边）→ 查询时优先返回
- $0.5 \leq W \leq 0.8$：中等关联（黄色边）→ 作为背景知识
- $0.3 \leq W < 0.5$：弱关联（灰色边）→ 仅在深度查询时返回

### 2.3 工作流

```
ASTRA内容提取完成（所有节点+初步边）
  ↓ 遍历所有节点对
对每对节点计算四个分量
  ↓ 组合加权
生成综合权重W
  ↓ 与阈值θ比较
W > θ ? 保留边 : 删除边
  ↓ 边权重标记
完成加权关系图
```

## 第三部分：动态掌握度模型

### 3.1 问题分析

**现状**：ASTRA本身没有学生学习跟踪能力，无法区分不同学生对知识的掌握程度。

**教学需求**：
- 追踪每个学生对每个知识点的掌握程度
- 识别薄弱环节，提供针对性复习建议
- 根据掌握度自适应调整内容返回的深度和范围
- 实现知识之间的"连锁反应"：掌握了前置知识→相关知识的掌握度自动提升

### 3.2 掌握度定义与初始化

#### 3.2.1 掌握度的多维表示

**单点掌握度** $M(k, s)$：
- $k$：知识点ID
- $s$：学生ID
- 取值范围：[0, 1]
  - 0.0-0.2：不了解（Haven't started）
  - 0.2-0.4：初步了解（Beginner）
  - 0.4-0.6：基本掌握（Intermediate）
  - 0.6-0.8：较好掌握（Proficient）
  - 0.8-1.0：熟练掌握（Expert）

**初始值**：$M(k, s) = 0.0$（新学生从0开始）

**持久化**：存储在图数据库的Node对象中
- 字段：`mastery_degree`
- 辅助字段：`mastery_last_updated`、`mastery_update_count`、`mastery_records`（历史记录）

### 3.2.2 评估类型与权重

**练习题评估** $(P)$：
- 权重：$\alpha_p = 0.3$
- 频繁但低风险的反馈
- 学生可多次尝试

**考试评估** $(E)$：
- 权重：$\alpha_e = 0.4$
- 权重比练习题高，反映正式评估的重要性
- 单次但高信度

**作业评估** $(A)$：
- 权重：$\alpha_a = 0.35$
- 介于两者之间

### 3.3 掌握度更新机制

#### 3.3.1 单点更新公式

**指数平滑移动平均**：

$$M_{new}(k, s) = \alpha \cdot Score + (1 - \alpha) \cdot M_{old}(k, s)$$

其中：
- $\alpha$：平滑因子（根据评估类型选择）
- $Score$：新评估的得分（0-1）
- $M_{old}$：旧掌握度
- $M_{new}$：新掌握度

**设计理由**：
- 给予新数据更多权重（体现学生最新状态）
- 保留历史信息（避免短期波动影响）
- 符合学习心理学中的"遗忘曲线"模型

**示例**：
- 某学生"导数"旧掌握度 = 0.4
- 参加练习评估，得分 = 0.75
- 新掌握度 = $0.3 \times 0.75 + 0.7 \times 0.4 = 0.225 + 0.28 = 0.505$ ✓

#### 3.3.2 级联更新策略

**问题**：单点更新后，相关知识点的掌握度也应随之调整
- 掌握了前置知识 → 后续知识学习效率提高 → 初始掌握度提升
- 掌握了核心概念 → 相关概念理解加深 → 掌握度提升

**级联更新算法**：BFS多层传播

```
输入：更新的知识点k、新掌握度M_new(k)
输出：所有受影响知识点及其新掌握度

1. 获取源节点k的最新掌握度：M_now = M_new(k)
2. 自动计算更新深度depth：
   - 若M_now < 0.3（掌握度低）：深度 = 3（广泛传播）
   - 若0.3 ≤ M_now < 0.6（掌握度中等）：深度 = 2
   - 若M_now ≥ 0.6（掌握度高）：深度 = 1（局部影响）
3. BFS遍历：
   - 初始化队列 = [(k, 深度0)]
   - 循环直到队列为空：
     a. 弹出节点k'及其当前深度d
     b. 若d >= depth，跳过
     c. 获取k'的所有邻接节点
     d. 对每个邻接节点n'：
        - 获取边权重w = edge_strength(k' → n')
        - 计算更新因子：factor = w × cascade_factor(0.5默认)
        - 计算新掌握度：M'_new = (M'_old + M_now × factor) / (1 + factor)
        - 考虑传播方向：
          * 若M_now < M'_old：向下传播降低（学生发现前置知识不扎实）
          * 若M_now > M'_old：向上传播提升（学生掌握新知识有助学习相关内容）
        - 更新n'的掌握度
        - 将(n', d+1)加入队列
   e. 继续BFS
```

**级联深度的含义**：

| 掌握度区间 | 深度 | 影响范围 | 解释 |
|-----------|------|---------|------|
| < 0.3 | 3 | 广泛 | 发现薄弱环节，需多管齐下加强 |
| 0.3-0.6 | 2 | 中等 | 正常学习进度，适度影响周边 |
| ≥ 0.6 | 1 | 局部 | 掌握扎实，主要影响直接相关概念 |

**示例**：
```
场景1：学生"微积分基础"掌握度从0.1→0.5（练习）
  - 新掌握度0.5，属于"中等"区间 → 深度2
  - 直接邻居（深度1）：导数、积分 → 掌握度各+0.1
  - 邻居的邻居（深度2）：极限、无穷级数 → 掌握度各+0.05

场景2：学生"导数计算"掌握度从0.5→0.75（考试）
  - 新掌握度0.75，属于"高"区间 → 深度1
  - 直接邻居：反向传播、链式法则 → 掌握度各+0.08
  - 不影响更深层级
```

### 3.4 自适应查询改进

**查询时的掌握度应用**：

```
用户查询："什么是梯度下降？"（学生s1）
↓
标准ASTRA检索 + 掌握度感知
↓
情况A：若前置知识掌握度低（< 0.4）
  → 自动添加前置知识上下文
  → 返回结构会是：
     [前置知识概述] → [梯度下降] → [应用案例]
  → 上下文深度提升20%

情况B：若前置知识掌握度高（≥ 0.6）
  → 跳过基础内容，直接进阶
  → 返回结构会是：
     [梯度下降进阶] → [优化变种] → [应用问题]
  → 上下文深度普通

情况C：若相关概念掌握度参差不齐
  → 智能推荐补习路径
  → 返回："建议先学习X，再学习Y"
```

### 3.5 工作流

```
学生完成评估（练习/考试/作业）
  ↓ 提取答题记录
计算单次评估得分
  ↓ 指数平滑更新
某知识点k掌握度：M_old → M_new
  ↓ 触发级联更新
自动决定传播深度（基于M_new）
  ↓ BFS多层传播
相邻知识点掌握度自动调整
  ↓ 持久化到数据库
完成一轮学习反馈循环
  ↓ 用户查询时
检索考虑掌握度信息
  → 自适应调整返回内容的组织方式
```

## 第四部分：集成设计

### 4.1 数据结构扩展

#### 4.1.1 节点(Node)字段扩展

| 字段名 | 类型 | 来源 | 说明 |
|-------|------|------|------|
| `entity_type` | enum | 结构设计 | "structural"\|"concept"\|"detail" |
| `structural_level` | int | 结构设计 | 0=Part, 1=Chapter, ..., 4+=Concept |
| `parent_structural_id` | str | 结构设计 | 上级节点ID（用于重建树） |
| `estimated_importance` | float | 元数据 | [0,1]，教材标注的重要性 |
| `mastery_degree` | float | 掌握度模型 | [0,1]，学生对该知识点的掌握度 |
| `mastery_last_updated` | str | 掌握度模型 | ISO 8601时间戳 |
| `mastery_records` | JSON | 掌握度模型 | [{timestamp, degree, type, score}, ...] |
| `mastery_update_count` | int | 掌握度模型 | 该知识点被更新过的次数 |
| `cascade_update_count` | int | 掌握度模型 | 被级联更新过的次数 |

#### 4.1.2 边(Edge)字段扩展

| 字段名 | 类型 | 来源 | 说明 |
|-------|------|------|------|
| `relation_strength` | float | 权重系统 | [0,1]，修改为精确计算值 |
| `edge_category` | enum | 权重系统 | "structural"\|"content" |
| `weight_components` | dict | 权重系统 | {sem_sim, struct_prox, co_occur, explicit} |
| `is_structural` | bool | 结构设计 | true表示来自元数据 |
| `from_metadata` | bool | 结构设计 | true表示来自元数据中的关系定义 |

### 4.2 模块交互

```
┌─────────────────────────────────────────┐
│         教学知识图谱系统架构              │
└─────────────────────────────────────────┘

输入层：
  ├─ 教材元数据 (JSON/YAML)
  ├─ 教材文本内容
  └─ 学生评估数据 (练习/考试)

第一层：结构层（自顶向下）
  ├─ 元数据解析与验证
  ├─ 分层节点创建（递归）
  ├─ 层级边构建
  └─ 前置关系映射
  
第二层：内容层（ASTRA标准流程）
  ├─ 文本分块
  ├─ 实体/关系提取
  ├─ 概念节点创建
  └─ 初步边创建

第三层：权重层
  ├─ 四分量权重计算
  ├─ 综合权重组合
  ├─ 权重阈值过滤
  └─ 边权重更新

第四层：掌握度层
  ├─ 学生评估输入
  ├─ 单点掌握度更新（指数平滑）
  ├─ 级联传播（BFS）
  └─ 掌握度记录持久化

查询层：
  ├─ 接收学生查询 + 学生ID
  ├─ 检索文本相似概念
  ├─ 考虑权重信息（强关联优先）
  ├─ 考虑掌握度（自适应深度）
  ├─ 组织返回结果
  └─ 可选推荐学习路径
```

### 4.3 与ASTRA的兼容性

**非侵入式设计**：
- ✅ 结构层完全独立：仅在插入前执行一次
- ✅ 权重层是补充：对ASTRA的边进行后处理
- ✅ 掌握度层是可选：不影响ASTRA的核心查询逻辑
- ✅ 查询时自适应：若无学生ID则回退到标准ASTRA行为

**扩展方式**：
```python
class StructureAwareASTRA(ASTRA):  # 继承ASTRA
    """教学特化版ASTRA"""
    
    # 新增方法
    def init_from_metadata(metadata)  # 初始化结构
    def compute_weights()              # 计算权重
    
    # 覆盖方法
    def query(question, student_id)    # 考虑掌握度的查询

    # 原有方法保持不变
    # 若用户不提供student_id，行为与ASTRA相同
```

### 5.2 优先级建议

**第一阶段（核心）**：结构层 + 权重层 [10天]
- 目标：构建完整的加权知识图
- 产出：StructureAwareASTRA基础类
- 意义：即使不考虑学生，权重已大幅提升查询质量

#### 6.2.2 应用场景

**场景1：新生学生第一次查询**
```

教学系统：
  返回：针对初学者设计的渐进式解释
  1. 先解释感知器（基础）
  2. 再解释神经元（单元）
  3. 最后解释网络拓扑（整体）
  → 学习体验顺畅 ✓
```

**场景2：学生参加中期测验**
```
学生在线测验：线性回归、梯度下降得分偏低

掌握度模型：
  - 线性回归：从0.0 → 0.35
  - 梯度下降：从0.0 → 0.28
  
级联更新后：
  - 矩阵运算：0.0 → 0.10（受负面影响）
  - 微积分基础：0.0 → 0.15（同上）
  
系统推荐：
  建议学生先补"微积分基础"，再深入"梯度下降"
  → 防止学生继续困境 ✓
```

**场景3：重点难点识别**
```
课程进行到第4周，教师检查系统：

  - 导致学生困难的原因：与链式法则关联度高但
    学生链式法则掌握度只有0.35
  
教师决策：
  下周加强链式法则的练习题，而不是直接讲反向传播
  → 科学地优化教学安排 ✓
```
